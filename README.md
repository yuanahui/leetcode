

# leetcode
#### commit信息规范：

- [feature|doc|bugfix\] {题目类别}\_{题目号},{题目类别}\_{题目号}, {yang|hui} 

- 说明：

  - feature|doc|bugfix

    - feature：新实现了题目
    - doc：添加了知识文档
    - bugfix：修改了之前实现的bug 或者优化了代码

  - 题目类别

    - 就是leetcode中题目的标签
    - 如位运算（用bite表示）、排序（sort）等

  - 题目号

    - 就是leetcode中题目的题目号
    - 如136，ms0507,offser15等

  - yang|hui

    - 实现的小伙伴的简称
    - yang-张春阳
    - hui-袁慧，也可以自己定

  - 备注：

    - 一次提交只能是feature|doc|bugfix中的一种：
    - 一次提交可包含以逗号(,)隔开的多个题目，如bite_10,bite_11
